# 보안 정책

## 보고 대상

다음과 같은 문제를 발견하셨다면 **공개 이슈로 올리지 말고** 아래 연락처로 먼저 알려 주세요.

- API 키·비밀번호·토큰이 저장소에 노출된 경우
- 리뷰 API를 통한 비정상적인 데이터 조작·삭제
- XSS, CSRF 등 웹 취약점
- 의존성 패키지 관련 알려진 취약점(CVE)

## 연락

- **공주대학교 지리학과** 임승연, 김혜린  
  (학교 이메일 또는 캡스톤 지도 교수님 안내 채널로 연락해 주세요.)

가능하면 재현 방법, 영향 범위, 관련 URL/화면을 함께 적어 주시면 조치에 도움이 됩니다.

## 저장소 운영 시 주의 (기여자용)

1. **`.env.local`**, 카카오 JavaScript 키, 개인 토큰은 **절대 커밋하지 마세요.**
2. 환경 변수 예시만 필요하면 **`.env.example`** 을 참고하세요.
3. `data/reviews.json` 등 운영 데이터를 올릴 때 개인정보(실명, 연락처)가 없는지 확인하세요.
4. Pull Request 전에 `npm audit` 결과를 확인해 주세요.

## 공개 일정

확인된 취약점은 심각도에 따라 수정 후, 필요 시 이슈/릴리스 노트에 요약을 공개할 수 있습니다.

## 범위 밖

- 카카오맵·호스팅(Vercel 등) 플랫폼 자체의 장애
- 제3자가 제공하는 데이터의 정확성
- 사용자 단말기·브라우저 확장 프로그램으로 인한 문제
